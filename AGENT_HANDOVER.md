## Current system state

### Live data chain (active)
```
EmotiBit hardware
  → EmotiBit Oscilloscope (C:\Program Files\EmotiBit\EmotiBit Oscilloscope\)
  → OSC UDP localhost:12345  (channel: /EmotiBit/0/HR)
  → python emomirror/osc_to_ws_bridge.py  ← must be running
  → WebSocket ws://localhost:8765
  → biosignal.js in browser
```

To start the bridge: `python emomirror/osc_to_ws_bridge.py`  
If bridge is not running, `biosignal.js` auto-falls back to mock silently.

### RSA metric
- `_target = 12` bpm swing (inhale HR peak − exhale HR trough, averaged over last 3 cycles)
- Win condition: RSA ≥ 12 bpm held for **30 continuous seconds** → `triggerBloom(false)`
- Timeout: 3 minutes without win → `triggerBloom(true)` (muted gold, "You showed up.")

## Known issues / next things to fix

### 1. HP bar visual — needs real-world calibration
The target of 12 bpm swing is a guess. With real EmotiBit data, check what swing values you actually get during the challenge and adjust `_target` in `ui_mockup.html` (search for `_target = 12`). A realistic achievable target for most people doing 4-4-6-2 breathing is 8–14 bpm.

### 2. Avatar re-parenting
On challenge start, `avatar-figure` + glitch clones are moved via `_adoptAvatar()` into `#breath-box-wrap`. On stop/bloom they are moved back via `_releaseAvatar()`. If the bloom scene avatar looks wrong (wrong size/position), check that `_releaseAvatar()` is being called before `triggerBloom` renders.

The CSS that controls avatar size inside the box is:
```css
#breath-box-wrap .anatomy-figure {
    position: absolute;
    top: 50%; left: 50%;
    height: 82%;
    width: auto;
    z-index: 1;
}
```
Adjust `height: 82%` to resize the avatar within the box.

### 3. RSA only activates after first complete breath cycle
The bar stays at 0 until the first full inhale→exhale cycle completes (needed to compute peak−trough). This is correct behavior but may feel like a slow start. If you want the bar to show something immediately, seed `_lastRSA` with a small value (e.g., 2) in `challengeEngine.start()`.

### 4. Box breathing dot — variable speed works but phase labels may lag
The peripheral countdown (small number next to active label) counts down from `phase.dur` each second. If the dot tween and the 1Hz ticker get out of sync after many cycles, the countdown may drift. Fix: reset `_phaseSecsLeft = phase.dur` precisely on `_advancePhase()` rather than relying on the ticker.

### 5. Demo mode (D key) still uses mock 'success' profile
If you press D while on the review scene, it starts biofeedback with the mock 'success' profile. To change the demo profile, find:
```js
biosignalSource.setMockProfile('success');
```
in the `keydown` handler in `ui_mockup.html`.

## How to run a full demo

1. Put on EmotiBit, open EmotiBit Oscilloscope, confirm HR signal is live
2. `python emomirror/osc_to_ws_bridge.py` — leave terminal open
3. Open `ui_mockup.html` in Chrome
4. Go through review scene → click "Start Biofeedback Challenge"
5. Breathe slowly with the dot. Bar fills as RSA swing grows. Hold green for 30s to win.
